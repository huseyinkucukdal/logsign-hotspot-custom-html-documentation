Getting Started


Welcome, Let's Build Your Hotspot Frontpage

You can design your hotspot prontpage by reading this documentation.You can find here all information you need. Just follow this documentation.

Let's begin with what you must know to able customize it.



Good To Know

Firts of all you should know that you only have a HTML page and you can not use any external CSS,JAVASCRIPT files or any link.Also because people will see the page as offline you must use BASE64 format images. To convert your images to BASE64 format you can use any BASE64 converter on internet.


Embedded Libraries

  • Twitter Bootstrap 3.0
  • jQuery
  • Font Awesome
  • Angular JS

Style

Because it is not possible to have external CSS file, CSS must be written in custom html field in <style> </style> tag.

Moreover, <style> </style> should be written at the top of DOM elements. By this way CSS will be be downloaded before rendering DOM so user will not have to see DOM without style. 

<style> </style>

Register Mode

Register mode is used to define form type. Every login form must have a register mode. Moreover it provides flexibility passage between forms.

It is used in ng-show attribute by equalizing form name like following example.

  • sign_up_sms
  • sms_verify
  • login
  • change_password
  • sign_up_citizenship_number
  • sign_up_sponsored
  • sign_up_cc
  • remember_password
  • first_login_change_password
  • sign_up_passport_number
  • user_info_page
<form ng-submit="sendSms()" name="sms_form" ng-show="portal.register_mode=='sign_up_sms' || portal.register_mode=='sign_up_passport_number'" novalidate >
...
</form>


Form Validation

It is possible to check validation for input elements and if input is valid submit button is active otherwise submit button is deactive.

For this proccess, ng-required is used at submit button. Also required attribute must be attached to inputs.

First, form name is written. Then .$invalid like following example. 

< button type="submit" ng-disabled="sms_form.$invalid">
{{'Send'|translate}}
</button>


Form Name

Every form tag (<form></form>) must have a name. Form names must be like following list.

  • sms_form
  • cn_form
  • sponsored_form
  • cc_form
  • verification_form
  • login_form
  • password_remember_form
  • first_login_change_password_form
  • change_password_form
<form ng-submit="sendSms()" name="sms_form" ng-show="portal.register_mode=='sign_up_sms' || portal.register_mode=='sign_up_passport_number'" novalidate>
...
</form>


Last Response

Last response is the response is gotten from server after user clicks to submit button. Last response gives information like about the resason user can not login. It is possible to display last response like following example. 

<div class="alert alert-success form_last_response" ng-show="portal.lastResponse">
  {{portal.lastResponse|translate}} 
</div>


ng-show

ng-show is used to display or hide form or some elements as pedepending on the parameter it gets. If value of paramet is true it will display otherwise hide.

ng-show is especially usefull for passage between forms. For example, when user clicks to submit button of sign_up_sms form user register is done and also login form is displayed. 

<div ng-show="portal.register_mode=='sign_up_sms' || !portal.register_mode">
  ...
<div>

<div ng-show="portal.lastResponse">
  {{portal.lastResponse|translate}}
</div>

!portal.register_mode is used to set first form user is faced.



ng-required

ng-reqired is used at inputs to check validation of inputs. 

<phonenumber-directive ng-cookie-value="form.username" model='portal.form.username' ng-required="portal.user_settings.login_type=='gsm_password'"> </phonenumber-directive>
 <input type="text" ng-model="portal.form.name" capitalize placeholder='{{"Name"|translate}}' ng-required="portal.user_settings.show_name_field"  class="form-control"> 
 <input type="text" ng-model="portal.form.lastName" capitalize placeholder='{{"Last Name"|translate}}' ng-required="portal.user_settings.show_last_name_field" > 
 <input type="text" ng-model="portal.form.birthYear" ng-maxlength="4" placeholder="1970" ng-required="portal.user_settings.show_date_of_birth_field" class="form-control" maxlength="4"> 
 <input type="number" ng-model="portal.form.citizenshipNumber" placeholder='{{"Citizenship Number"|translate}}' min="0" pattern="\d*" ng-required="portal.user_settings.show_tc_no_field" class="form-control"> 
 <input type="email" ng-model="portal.form.email" placeholder='{{"Email"|translate}}' ng-required="portal.user_settings.show_email_field">


ng-model

portal.form object is already defined at controller. By using ng-model, data of user is created on portal.form base object. 

<input type="password" ng-model="form.password" />
<input type="text" ng-model="form.name" />


Place Holder

On placeholder attribute of inputs it is possible to use translation thanks to angularjs. by this way placeholder of inputs has language support. 

<input type="password" ng-model="changePasswordForm.password" placeholder="{{'Password'|translate}}" required class="form-control"/>



Functions


changeLocal( )

To change the UI language just pass the language code via changeLocal( ) function. 

<button ng-click="changeLocal('tr_TR');">Türkçe</button>
<button ng-click="changeLocal('en_US');">English</button>
<a href="#" ng-click="changeLocale('tr_TR');">{{"Turkish"|translate}}</a>
<a href="#" ng-click="changeLocale('en_US');">{{"Turkish"|translate}}</a>
